Usa el agente nestjs-xp-architect para elaborar un plan detallado sobre los siguientes requisitos:

$ARGUMENTS

El plan debe incluir:
1. Análisis de cómo la funcionalidad encaja en la arquitectura hexagonal existente
2. Identificación de Value Objects, Entidades y Agregados necesarios
3. Definición de puertos (interfaces) y adaptadores requeridos
4. Casos de uso a implementar siguiendo CQS
5. Estrategia de testing (qué tests escribir primero siguiendo TDD/BDD)
6. Orden de implementación siguiendo vertical slicing
